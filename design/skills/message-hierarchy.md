# Message hierarchy & page CRO

Runtime fragment: `MESSAGE_HIERARCHY` in `convex/lib/design/skills.ts`.

## Decide before writing

For every page: the single primary promise (the hero), 2-4 supporting
benefits (each must survive "so what?"), the proof, the objections to
answer, and the ONE action the page asks for. One page = one primary
action.

## Page order is a narrative, not a feature list

Hero (promise) → proof → the reader's problem in their own words → how it
works (exactly 3 steps: lower the barrier → the product working → the
outcome) → 3-5 benefits max → objection-matched proof → final CTA with
risk reversal. A Hero → Feature 1 → Feature 2 → Feature 3 → CTA page is a
list, not an argument.

## Hero rules

- Headline ≤10 words (6-8 ideal), containing the audience or their outcome.
- Subheadline: "{product} helps {audience} {outcome} by {mechanism}" plus
  a proof point or timeframe, in 1-2 sentences.
- CTA = action verb + what they get, first person ("Start my free trial",
  never Submit / Sign Up / Learn More / Click Here), with risk-reversal
  micro-copy beside it ("Free to start · no credit card").

## CRO audit — fixed impact order

1. Value-prop clarity (5-second test: what is it, why care).
2. Headline (specific, matches where the visitor came from).
3. CTA placement/copy/hierarchy (one primary, visible without scrolling,
   repeated at decision points).
4. Scannability (scanning alone yields the message).
5. Trust signals — placed near CTAs and after benefit claims.
6. Objection handling (price, "will it work for me", effort, risk).
7. Friction (form fields, unclear next steps, nav, mobile, load time).

Checkable gates: proof within one scroll of the first CTA; risk reversal
adjacent to every CTA; benefit-first headlines; urgency only where
authentic.

## Copy-to-design alignment

Visual hierarchy must reinforce the message hierarchy: the promise gets
the largest type; the primary CTA gets the strongest contrast; proof sits
next to the claim it supports; nothing decorative competes with the
argument. If the loudest element on the page isn't the primary message,
the page is misaligned.
