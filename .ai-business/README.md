# .ai-business — Brandkite growth control system

Operational memory for the autonomous effort to grow Brandkite to $1M ARR.
Any AI session (or human) should be able to resume from these files without conversation history.

**Start here → [STATE.md](STATE.md)** (canonical restart point), then [AUTHORIZATION.md](AUTHORIZATION.md).

| File | Purpose |
|---|---|
| STATE.md | Current state, constraint, next action — the restart point |
| ACCESS.md | Which systems are available/missing (no secrets) |
| AUTHORIZATION.md | Autonomy boundaries, budgets, approval rules |
| STRATEGY.md | Diagnosis, revenue models, chosen strategy |
| VISION.md | Long-range "idea → launch-ready company" direction |
| METRICS.md | Measurement framework, event taxonomy, baselines |
| ROADMAP.md | 90-day phased plan |
| BACKLOG.md | Ranked backlog (impact × confidence ÷ effort) |
| EXPERIMENTS.md | Experiment log with hypotheses and outcomes |
| DECISIONS.md | Dated decision log with rationale |
| LEARNINGS.md | Validated learnings about users/market/product |
| RISKS.md | Known risks and mitigations |
| CHANGELOG.md | Business-relevant shipped changes |
| RUNBOOK.md | Commands, deploy, ops procedures |
| sessions/ | Per-session work records (YYYY-MM-DD-HHMM.md) |

Conventions: keep files short and current; move stale detail into sessions/; never store secret values anywhere in this tree.
