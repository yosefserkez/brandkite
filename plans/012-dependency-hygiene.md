## Why this matters

The manifest carries dead weight and one security exposure:
- **Unused deps** (`recharts`, `pdf-parse`, `@ai-sdk/anthropic`, `atmn`) inflate
  install size and audit noise and mislead future work (someone will assume
  charts/anthropic/atmn are wired up).
- **Duplicate animation library**: both `framer-motion` and `motion` are declared
  (same engine); `framer-motion` is imported by exactly one file while `motion`
  is used by ~5. Shipping both risks two copies of the animation runtime.
- **Undeclared dep**: `scripts/logos/makeZip.ts` imports `yazl` which is not in
  `package.json` (resolves only via hoisting today; breaks under strict/CI
  installs — one of the current `tsc` errors).
- **Critical advisory on a runtime path**: `fast-xml-parser` (entity-encoding
  bypass) reaches production via `@convex-dev/r2 > @aws-sdk/* > fast-xml-parser`
  (R2 parses S3 XML responses — used for every logo/asset URL). A `pnpm.overrides`
  pin closes it.
- **Dead component**: `src/components/shadcn-studio/` (one orphaned file, no
  references).
