## Direction findings (options for the maintainer ‚Äî not ranked against bugs)

Grounded in `.ai-business/V2-DIRECTION.md` and the schema. Not turned into plans
(the operator selected fix-plans); listed so the direction work is captured.

- **Custom domains for published sites** ‚Äî V2-DIRECTION names "publish the hosted
  site (custom domain)" as the Pro-tier justification, but `convex/site.ts` only
  supports slugs under `/s/:slug`. This is the single biggest gap between the
  stated Pro value prop and the code. A design/spike plan (DNS/verification,
  where custom-domain routing terminates on Vercel + Convex) is the natural next
  step if monetization is the focus.
- **Brand-as-API / design tokens export** ‚Äî V2-DIRECTION's "scale/infra" layer
  and DESIGN-ENGINE.md Phase C. The data model (one source-of-truth per module)
  already supports it; `export.ts` is one route from a token/JSON API. A spike
  to define the token schema + a read API surface would open the "Stripe-of-brand"
  positioning.
- **Team invites** ‚Äî `companyMembers` (owner/editor/viewer) exists in the schema
  and is honored in access checks, but there is no invite flow UI. Low-effort
  activation/expansion lever the data model already supports (BACKLOG #10).
